# 🤖 AI_GUIDELINES — правила доработки проекта через ИИ

Этот документ регламентирует, как AI-ассистенты (ChatGPT, Claude, Copilot и др.) должны вносить изменения в проект Mini ERP. **Соблюдение этих правил обязательно** для сохранения работоспособности и качества кода.

---

## ✅ Допустимые изменения

### 1. Добавление новых полей в сущности
- В DTO (`dto/Requests.kt`, `dto/Responses.kt`)
- В Entity (`domain/*.kt`)
- **Обязательно**: одновременно обновить метод `from()` в соответствующем Response-классе
- **Обязательно**: добавить поле в `DataInitializer.kt`, если оно обязательное

### 2. Добавление новых REST endpoints
- Только в существующих контроллерах (`controller/ErpController.kt`)
- С соблюдением REST-стандартов (правильные HTTP-методы, коды ответов)
- С валидацией через `@Valid` и Jakarta-аннотации
- **Обязательно**: покрыть тестами в `controller/`

### 3. Добавление новой бизнес-логики
- Только в `service/` слое
- Через новые методы или новые сервисы
- **Обязательно**: покрыть юнит-тестами в `service/`

### 4. Добавление новых сущностей
Создать полный набор файлов:
- Entity в `domain/`
- Repository в `repository/`
- Service в `service/`
- Endpoints в `ErpController`
- DTO в `dto/`
- Тесты для Service и Controller
- HTML-страницу в `static/`

### 5. Исправление багов
- С обязательным добавлением **регрессионного теста**, воспроизводящего баг
- Тест должен падать до исправления и проходить после

## ⚠️ Зоны повышенного риска

### 🔴 КРИТИЧНО: Логика статусов WorkOrder

**Файл**: `service/WorkOrderService.kt`, метод `updateStatus()`

**Правила переходов**:
- `CREATED → IN_PROGRESS → COMPLETED` — нормальный путь
- `CREATED → CANCELLED` или `IN_PROGRESS → CANCELLED` — допустимо
- `COMPLETED → *` и `CANCELLED → *` — **ЗАПРЕЩЕНО**
- При `COMPLETED` оборудование возвращается в `ACTIVE`
- При создании наряд-заказа оборудование переходит в `MAINTENANCE`

**Любое изменение этой логики требует**:
1. Обновления `ARCHITECTURE.md` (раздел "Жизненный цикл наряд-заказа")
2. Добавления тестов на все новые переходы
3. Проверки, что `completedAt` заполняется корректно

### 🔴 КРИТИЧНО: JPA Relationships

**Файлы**: `domain/WorkOrder.kt`, `domain/Equipment.kt`

**Правила**:
- `@ManyToOne(fetch = FetchType.LAZY)` — ленивая загрузка
- При сериализации в JSON **обязательно** использовать DTO, иначе:
  - `LazyInitializationException` (если сессия закрыта)
  - Бесконечная рекурсия (если добавить двусторонние связи)
- **Никогда не возвращать Entity напрямую из Controller**

### 🔴 КРИТИЧНО: Spring Boot 4.x

**Важно**: Проект использует **Spring Boot 4.1.0** — новую мажорную версию.

**Особенности Spring Boot 4.x**:
- Требует **Jakarta EE 11** (не javax)
- Возможны изменения в API по сравнению с 3.x
- Некоторые устаревшие методы могут быть удалены
- Требуется **JDK 26+**

**При обновлении зависимостей**:
1. Проверить совместимость с Spring Boot 4.1.0
2. Выполнить `mvn clean install -U`
3. Проверить, что все тесты проходят
4. Проверить логи на наличие deprecation warnings

### 🟡 ВАЖНО: Форматирование ktlint

**Файлы**: все `.kt` файлы

**Правила ktlint 1.8.0**:
- ❌ Запрещены wildcard-импорты (`import com.minierp.dto.*`)
- ✅ Обязательны trailing commas в многострочных списках
- ✅ Импорты в алфавитном порядке
- ✅ Нет пустых строк сразу после `class X {`
- ✅ Файл заканчивается переводом строки

**Перед коммитом всегда**: `mvn spotless:apply`

### 🟡 ВАЖНО: Тесты с MockK

**Файлы**: `src/test/kotlin/com/minierp/...`

**Правила**:
- Обязательна аннотация `@ExtendWith(MockKExtension::class)` у тестового класса
- Мок `save()` должен возвращать объект с присвоенным `id`, иначе `WorkOrderResponse.from()` упадёт с NPE
- Использовать `Optional.of(...)` для `findById()`
- **Не смешивать** MockK и Mockito в одном проекте

### 🟡 ВАЖНО: Сборка WAR

**Файл**: `pom.xml`

Проект собирается в **`mini-erp.war`** (не JAR). Это имя зафиксировано в `maven-war-plugin`:

```xml
<plugin>
    <groupId>org.apache.maven.plugins</groupId>
    <artifactId>maven-war-plugin</artifactId>
    <configuration>
        <failOnMissingWebXml>false</failOnMissingWebXml>
        <warName>mini-erp</warName>
    </configuration>
</plugin>
